# ERP 및 업무 시스템 분석 사례 (익명화)

ERP 문제는 데이터가 아니라  
**구조와 관계의 문제**인 경우가 많습니다.

이 문서는 ERP 데이터를  
Knowledge Graph 관점에서 재해석한 사례입니다.

---

## 문제 유형

- 여러 요소가 결합된 결과 수치
- 기간별 변화 원인 분석
- 규칙과 계산식의 복잡성
- 설명 요구가 높은 의사결정

---

## 적용 구조

- ERP 데이터 → ABox
- 도메인 개념 → Ontology (TBox)
- 계산/배부 규칙 → 지식 구조화
- Graph 탐색으로 원인-결과 추적

---

## 효과

- 구조 기반 분석 가능
- “왜 이 값이 나왔는지” 설명 가능
- 도메인 지식의 재사용

---

## 요약

- ERP는 조회 시스템이 아니다
- KG는 ERP 위의 의미 계층이다

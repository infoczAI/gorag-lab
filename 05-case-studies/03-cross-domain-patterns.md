# 도메인을 넘는 공통 패턴

문서, ERP, 규칙 문제는  
겉보기에는 전혀 다른 문제처럼 보입니다.

하지만 Knowledge Graph 관점에서는  
공통된 패턴을 가집니다.

---

## 공통점

- 개념 정의가 중요하다
- 관계가 결과를 만든다
- 규칙과 제약이 핵심이다
- 설명 가능성이 요구된다

---

## 반복되는 구조

- Ontology → 기준
- Knowledge Graph → 구조
- Agent → 탐색
- LLM → 설명

이 패턴은 도메인에 독립적입니다.

---

## 요약

- KG는 특정 도메인 기술이 아니다
- 문제 구조를 다루는 방식이다
- 도메인이 바뀌어도 패턴은 유지된다
